# 首轮学生内测执行包

## 对象与任务

邀请 3-5 名高中生，每人使用自己最近的一张月考或阶段测试卷。任务只说一句：“把这张卷子导入，做至少 3 道题，并把其中一道错题完成订正和重做。”过程中不教学、不提示按钮位置，只记录卡点。

## 每人记录

- 是否成功注册与登录
- PDF 是否成功拆题，首次可做题用了多久
- 是否完成一次独立作答和 AI 评阅
- 是否相信评阅，哪一句让他信或不信
- 是否完成订正和遮蔽重做
- 明天是否愿意再次打开，为什么

## 访谈三问

1. 导入过程最卡的地方在哪里？
2. 哪一次评阅你相信或不相信？为什么？
3. 明天你会因为什么再次打开 Studyoo？

## 数据导出

在生产后端环境运行：

```bash
npm run report:funnel --workspace backend
```

Railway 生产环境可从项目目录执行：

```bash
ssh-add ~/.ssh/id_ed25519
railway ssh -i ~/.ssh/id_ed25519 npm run report:funnel --workspace backend
```

不要在访谈笔记里记录学生密码、完整联系方式或试卷中的个人身份信息。
