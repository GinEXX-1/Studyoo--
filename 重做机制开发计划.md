# 重做机制开发计划

制定日期：2026-07-12
状态：待评审（GinEXX 确认后排期开发）

---

## 为什么需要重做机制

Studyoo 的核心主张是"订正而不是跳过思考"。但当前链路在评阅后就断了：

- 学生提交作答 → AI 评阅 → **看完反馈就结束**，没有"合上解析重新做一遍"的入口。
- `review_tasks` 间隔复习（当天/3/7/14 天）已经存在，但它复习的是**同类题**，缺少"原题重做验证订正效果"这一环。
- 评阅永远是"第一次视角"，AI 不知道学生上次错在哪，无法给出"这次比上次进步在哪"的反馈。

重做机制补上这一环：**错题 → 订正 → 遮蔽旧作答重做原题 → AI 对比评阅 → 掌握度升级**。

---

## 状态机设计

一道做错的题，从"错"到"会"经历以下状态（记录在题目粒度）：

```
wrong（评阅未通过）
  → corrected（学生看过反馈/解析，标记"我已订正"）
  → redo_pending（等待重做，建议与订正间隔 ≥ 30 分钟）
  → redo_passed（重做通过 → 触发间隔复习链，通往 mastered）
  → redo_failed（重做仍错 → 回到 corrected，最多循环 3 次后建议求助解析入口）
```

与现有 `mastery_status`（weak/reviewing/mastered）的关系：redo_passed 把错题记录从 weak 升到 reviewing；后续间隔复习全对才升 mastered。**重做通过 ≠ 掌握**，这是刻意设计。

---

## 数据库改动（最小方案）

不建新表，扩展 `practice_attempts`：

| 新列 | 类型 | 说明 |
|---|---|---|
| `attempt_round` | INTEGER DEFAULT 1 | 第几次作答（重做 = 上一次 round + 1） |
| `parent_attempt_id` | TEXT | 指向被重做的那次作答 |
| `is_redo` | INTEGER DEFAULT 0 | 是否重做（便于统计） |

`mistake_records` 加一列 `correction_status`（wrong/corrected/redo_pending/redo_passed）承载状态机。

## API 设计（遵循现有契约风格）

### 1. 标记已订正
`POST /practice/questions/:questionId/corrections`
请求：`{ "note": "我理解了：应取第 3 个数而不是平均值" }`（订正笔记可选但强烈建议，一句话即可）
响应：`{ correction_status: "redo_pending", redo_available_at: "..." }`

### 2. 获取重做题面（遮蔽模式）
`GET /practice/questions/:questionId?mode=redo`
与普通题面相同，但**不返回** official_answer_text、上次 attempt、上次 feedback——前端也不得从缓存展示。

### 3. 提交重做
`POST /practice/questions/:questionId/attempt`（复用现有接口）
请求增加：`{ "answer_text": "...", "redo_of_attempt_id": "<上次attempt id>" }`
后端行为差异：
- 评阅 prompt 附带上一次作答与上一次的缺口分析，要求 AI 输出 `progress_note`（这次比上次进步/仍卡在哪一步）。
- 通过（score ≥ 80）→ correction_status = redo_passed，mastery_status 升 reviewing，触发 `createReviewTasks`。
- 不通过 → redo_failed 回 corrected，redo_count + 1。

### 4. 订正历史
`GET /practice/questions/:questionId/attempts` → 按 round 排列的作答记录，用于"理解成长轨迹"展示（呼应 README 的 Review 能力）。

## 前端改动

1. **PracticePage 评阅卡片**底部增加两个按钮：「我已订正，稍后重做」（调 corrections 接口）/「立即重做」。
2. **重做页面** = 现有做题页 + 顶部黄色横幅"重做模式：上次得分已隐藏，独立完成后再对比"。
3. **对比反馈卡片**：分数并排（上次 62 → 这次 85）+ AI 的 progress_note。
4. **个人页**新增两个统计：订正率（错题中标记订正的比例）、重做通过率。

## 验收标准（写进 验收题集.md）

1. 做错验收题 1 → 标记订正 → 重做页面看不到上次作答与参考答案。
2. 重做答对 → 反馈里能看到与上次的对比；错题状态变为 reviewing；`GET /review/pending` 出现 4 轮复习任务。
3. 重做仍错 → 状态回 corrected，第 3 次失败后界面提示进入解析入口。
4. `GET /practice/questions/:id/attempts` 返回完整 round 序列。

## 设计红线

- 重做模式下，后端绝不下发参考答案与历史反馈（防前端绕过）。
- AI 对比评阅只在重做时启用，避免普通评阅 prompt 膨胀。
- 「立即重做」允许存在，但默认引导"稍后重做"（间隔效应）。
