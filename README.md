# **Studyoo（学有）**

**A Learning OS built for Chinese high school students.**

>   **Understanding belongs to you.**

---

## **What is Studyoo?**

Studyoo（中文名：学有）是一款专为中国高中生设计的 Learning OS（学习操作系统）。
我们并不希望成为另一款 AI 学习软件，也不希望成为另一个搜题平台、网课平台或聊天机器人。Studyoo 希望重新定义学生与知识、思考以及 AI 的关系。我们相信： **真正的学习，不是获得更多答案，而是建立更深的理解。** AI 不应该替学生完成思考，AI 应该帮助学生继续思考。因此，在 Studyoo 中，AI 始终位于系统底层，而学生始终站在学习过程的中心。

---

# **Why**

今天的中国高中生并不缺少学习资源。教辅、试卷、网课、搜题软件、AI 聊天机器人每天都在产生更多内容。
真正稀缺的，是理解。

很多学生都会经历这样的场景：拍一道题。得到答案。看完解析。依然不知道为什么想到这种方法。官方解析通常十分简洁，视频课程往往需要二十分钟甚至更长，普通 AI 又倾向于直接给出完整答案。
学生不断得到新的答案，却很少真正建立属于自己的理解。Studyoo 希望改变这一点。
我们希望 AI 成为学生思考过程中的辅助工具，而不是替代者。

---

# **Our Philosophy**

Studyoo 的设计建立在几个核心原则之上。

## **Understanding First**

理解始终优先于效率。

任何功能，都必须首先帮助学生建立理解，而不是帮助学生更快完成题目。

---

## **Parse, Don’t Answer**

Studyoo 的核心能力不是解题，而是解析。

解析包括：

- 解析题目
- 解析官方答案
- 解析知识点
- 解析思路
- 解析命题意图
- 解析学生的认知状态

相比于告诉学生”怎么做”，Studyoo 更关心”为什么这样做”。

---

## **AI Never Replaces Thinking**

AI 可以解释，可以引导，可以追问，可以提醒，但不能替学生完成思考。整个产品会尽量避免无限聊天。AI 的职责是帮助学生继续思考，而不是结束思考。

---

## **Built for Chinese High School Students**

Studyoo 只专注一个目标：帮助中国高中生建立真正的理解。整个产品围绕中国高中课程体系、高考命题方式以及中国学生的学习习惯进行设计。

---

# **Core Capabilities**

Studyoo 按照能力（Capability）设计，而不是按照页面设计。页面会不断变化，能力不会。目前，Studyoo 定义了七个核心能力。

## **Parse**

解析能力。

整个产品最重要的入口。

负责解析：

- 题目
- 图片
- 官方答案
- 知识点
- 思路
- 命题意图

Parse 是整个 Learning OS 的核心。

---

## **Memory**

学习记忆。

Memory 保存的不是聊天记录。

而是学习历史。

例如：

- 曾经解析过哪些题目
- 曾经建立了哪些理解
- 哪些地方仍然存在疑惑
- 理解是如何逐渐形成的

Memory 是 Studyoo 的长期记忆。

---

## **Cognition**

认知模型。

Studyoo 希望逐渐建立每一位学生自己的认知画像。

例如：

- 哪些知识已经熟练掌握
- 哪些知识容易遗忘
- 哪些思维方式仍然薄弱
- 哪些错误经常重复出现

未来，每一次 AI 解析都会参考认知模型，而不是只参考题目。

---

## **Knowledge**

知识图谱。

Studyoo 会维护三层知识结构：

课程知识树。

考试知识树。

学生个人知识树。

课程知识树负责描述课程体系。

考试知识树负责描述高考命题。

学生知识树负责描述个人掌握情况。

---

## **Decision**

学习决策。

Studyoo 希望减少学生每天思考”今天应该学什么”所花费的时间。

系统会结合学习历史、认知模型以及知识图谱，生成当天最值得完成的学习任务。

---

## **Review**

理解回顾。

Studyoo 回顾的不是错题数量。

而是理解的发展过程。

未来，学生可以看到：

第一次理解。

第一次独立应用。

第一次主动想到这种方法。

Studyoo 希望记录理解成长的轨迹。

---

## **Workspace**

学习工作台。

Workspace 是整个学习过程发生的地方。

它负责连接解析、记录、笔记、知识图谱以及学习历史。

未来，它会成为学生每天学习时最主要的工作空间。

---

# **User Journey**

Studyoo 的学习流程非常简单。

学生打开 App。

点击「解析题目」。

拍摄一道题目，或者粘贴题目、官方答案。

系统首先使用 OCR 将图片转换为文字。

随后结合知识库，对题目进行解析。

AI 不会立即输出一整篇答案。

Studyoo 首先生成整道题的思维路线。

学生可以快速了解这道题真正的突破口。

随后，每一个关键步骤都会展开解释。

每一步都可以继续深入追问。

每一步都可以记录自己的理解。

每一步都可以标记”仍然没有理解”。

整个学习结束以后，后台会自动更新学习历史、知识图谱以及认知模型。

学生最终获得的不是一份聊天记录。

而是一份不断成长的解析文档。

# **Technical Architecture**

Studyoo 希望保持工程上的简单与稳定。

因此，所有重要技术都会采用成熟方案。

## **Database**

数据库可以理解成 Studyoo 的长期记忆。

它负责保存：

- 用户信息
- 学习历史
- 解析记录
- 用户笔记
- 知识图谱
- AI 输出
- 认知模型

数据库不会思考。

它负责长期保存信息。

---

## **API**

API 可以理解成一条通信线路。

Studyoo 通过 API 与 AI 模型通信。

学生上传题目。

Studyoo 将请求发送给 AI。

AI 返回解析结果。

API 只负责传递数据。

它本身不会进行推理。

---

## **OCR**

OCR（Optical Character Recognition）负责识别图片中的文字。

当学生拍摄一道题时，系统首先通过 OCR 提取文字内容。

之后才会进入 AI 解析阶段。

OCR 更擅长识别文字。

AI 更擅长理解文字。

二者分别完成最擅长的工作。

---

## **Cache**

缓存（Cache）可以理解成便签。

如果一道题已经解析过。

系统可以直接读取已有结果。

这样能够减少等待时间，也能降低 AI 调用成本。

---

## **RAG**

RAG（Retrieval-Augmented Generation）是一种”先查资料，再回答问题”的工作方式。

在 Studyoo 中，AI 不会完全依赖自己的记忆。

解析开始之前，系统会先从知识库中检索：

- 课程标准
- 高考知识结构
- 官方教材
- 历年真题
- 项目维护的知识图谱

随后，将这些信息连同题目一起发送给 AI。

因此，Studyoo 更相信可验证的知识，而不是模型自身的记忆。

# **Development Principles**

整个项目遵循 README Driven Development。

任何重要产品方向变更之前，都应该首先更新 README。

文档分工如下：

- README 是产品愿景、定位、路线图的可信来源。
- `AI学习网站 API契约 v1.0.md` 是前后端接口的唯一真相来源。
- `PROGRESS.md` 是当前实现状态的唯一真相来源。

数据库、API、UI 与代码，都应该围绕 README 展开。

如果 README 与代码发生冲突，应优先检查代码是否已经偏离设计目标。

# **Local Development**

当前代码雏形采用：

- 前端：Vite + React，默认运行在 `http://localhost:5173`
- 后端：Node.js + Express + SQLite，默认运行在 `http://localhost:3000/api/v1`

启动前需要安装依赖：

```bash
npm install
```

启动后端：

```bash
npm run dev:backend
```

启动前端：

```bash
npm run dev:frontend
```

AI 相关接口需要在 `backend/.env` 中配置 `AI_API_KEY`。未配置时，接口会返回 `AI_SERVICE_ERROR`，不会使用假数据模拟 AI 结果。

Studyoo 希望有一天，当学生结束一天学习时，看见的不再是：

今天完成了多少题。

而是：

今天，你真正理解了什么。

学习从来不是不断积累答案。

学习是一棵树不断生长。

Studyoo 希望陪伴每一位中国高中生，建立属于自己的理解，建立属于自己的学习系统。
