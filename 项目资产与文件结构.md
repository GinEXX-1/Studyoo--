整理日期：2026-07-12

## 目录结构（整理后）

```
Studyoo/
├── backend/                  # Node.js + Express + SQLite（node:sqlite）
│   ├── src/
│   │   ├── server.js         # 入口：中间件、静态目录(需登录)、路由挂载
│   │   ├── config.js         # 配置（数据库/上传路径已锚定到 backend 目录）
│   │   ├── db.js             # 建表、种子数据、标签归一化、todayLocal
│   │   ├── migrate-v2.js     # v2 迁移（导入流水线/复习表；含 base64 清理）
│   │   ├── auth.js           # JWT 鉴权中间件
│   │   ├── quota.js          # AI 每日配额（先扣后调、失败回滚）★所有 AI 调用必须经过
│   │   ├── ai.js             # 所有 AI prompt 与调用（拆题/单题识别/评阅/组卷…）
│   │   ├── rate-limiter.js   # 认证接口限速
│   │   ├── routes.js         # 主路由（账号/真题/练习/答疑/错题/学习路径）
│   │   ├── routes-import.js  # PDF 结构化导入流水线（含单题裁切）
│   │   ├── routes-review.js  # 间隔复习调度
│   │   ├── routes-recommend.js # 规则驱动推荐
│   │   └── seed-*.js         # 内置题库种子
│   ├── scripts/
│   │   ├── smoke-minimal.js  # npm run smoke:minimal 最小链路冒烟
│   │   ├── test-ownership.mjs # npm run test:ownership 越权隔离测试（14 项）
│   │   └── check-ai.js       # AI 连通性检查
│   ├── uploads/              # ★正式上传目录（原卷 PDF/页面图/单题裁切图）
│   ├── studyoo.db            # ★正式数据库（唯一正库）
│   ├── studyoo-smoke.db      # 冒烟测试库（可随时删，会自动重建）
│   └── .env                  # 密钥配置（已 gitignore，勿提交）
├── frontend/                 # Vite + React
│   ├── src/pages|components|lib
│   └── dist/                 # 构建产物（已 gitignore）
├── *.md                      # 文档（见下方文档索引）
├── studyoo.db  ⚠️ 见"遗留资产"
├── uploads/    ⚠️ 见"遗留资产"
└── tmp/        ⚠️ 见"遗留资产"
```

## 遗留资产（需要 GinEXX 决定）

**背景**：修复前数据库/上传路径是相对启动目录的，从仓库根目录启动过服务，产生了第二套数据。2026-07-12 已修复（`config.js` 把路径锚定到 backend/），**今后无论从哪里启动都只会用 backend/ 这一套**。

| 遗留文件 | 内容 | 建议 |
|---|---|---|
| 根目录 `studyoo.db`（~17MB） | 6 个用户（多为测试号 + 一个"李俊豪"，7-12 注册）、4 个导入任务 | ⚠️ 如果你 7-12 白天在应用里做的数据现在"消失"了，说明它在这个库里——先别删，告诉 Claude 帮你合并；确认无需要的数据后可删除 |
| 根目录 `uploads/`（~40MB） | 上述分裂会话的 PDF/页面图 | 同上，与根目录 db 一起处置 |
| 根目录 `tmp/` | 临时 PDF | 确认后可删 |
| `backend/studyoo-smoke.db` | 冒烟测试产物 | 可随时删 |

**正库已瘦身**：backend/studyoo.db 通过清理 base64 冗余 + VACUUM，从 24MB 降到 1.1MB。
