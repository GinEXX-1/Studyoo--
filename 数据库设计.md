# Studyoo 数据库设计

第一版使用 SQLite 作为本地开发数据库，后续可以迁移到 PostgreSQL 或 Supabase。字段设计严格对齐 `AI学习网站 API契约 v1.0.md`。

## users

保存账号信息。

|字段|类型|说明|
|---|---|---|
|id|TEXT PK|用户 UUID|
|nickname|TEXT UNIQUE|昵称，第一版全局唯一|
|password_hash|TEXT|密码哈希，禁止保存明文密码|
|grade|TEXT|高一 / 高二 / 高三|
|subjects_json|TEXT|学科数组 JSON|
|created_at|TEXT|ISO8601|

## questions

保存一次提问或一道题。

|字段|类型|说明|
|---|---|---|
|id|TEXT PK|题目 UUID|
|user_id|TEXT FK|所属用户|
|subject|TEXT|学科|
|mode|TEXT|`solve_from_scratch` 或 `deepen_official_answer`|
|content_text|TEXT|题目文字|
|content_image_url|TEXT NULL|第一版固定为 NULL|
|official_answer_text|TEXT NULL|官方答案文字|
|official_answer_image_url|TEXT NULL|第一版固定为 NULL|
|knowledge_tags_json|TEXT|知识点标签数组 JSON|
|status|TEXT|answered / pending / failed|
|created_at|TEXT|ISO8601|

## answers

保存 AI 讲解。

|字段|类型|说明|
|---|---|---|
|id|TEXT PK|讲解 UUID|
|question_id|TEXT UNIQUE FK|对应题目|
|hint_text|TEXT NULL|从零开始问模式下的思路引导|
|step_breakdown_json|TEXT|分步骤解释数组 JSON|
|full_solution_text|TEXT NULL|完整解答|
|revealed_full_solution|INTEGER|0/1，是否已主动看完整答案|
|created_at|TEXT|ISO8601|

## follow_ups

保存追问记录。

|字段|类型|说明|
|---|---|---|
|id|TEXT PK|追问 UUID|
|question_id|TEXT FK|对应题目|
|user_id|TEXT FK|所属用户|
|content_text|TEXT|追问内容|
|reply_text|TEXT|AI 回复|
|created_at|TEXT|ISO8601|

## mistake_records

保存错题本条目。第一版规则：用户主动点击“看完整答案”即计入。

|字段|类型|说明|
|---|---|---|
|id|TEXT PK|错题记录 UUID|
|user_id|TEXT FK|所属用户|
|question_id|TEXT FK|对应题目|
|knowledge_tags_json|TEXT|知识点标签数组 JSON|
|mistake_count|INTEGER|计入次数|
|mastery_status|TEXT|weak / reviewing / mastered|
|last_reviewed_at|TEXT NULL|最后复习时间|

## learning_path_items

保存学习路径推荐。

|字段|类型|说明|
|---|---|---|
|id|TEXT PK|推荐 UUID|
|user_id|TEXT FK|所属用户|
|knowledge_tag|TEXT|知识点|
|reason|TEXT|推荐原因|
|recommended_action|TEXT|review / practice|
|related_question_ids_json|TEXT|关联题目 UUID 数组 JSON|
|status|TEXT|pending / done / dismissed|

## ai_usage

保存每日 AI 调用次数，用于限流。

|字段|类型|说明|
|---|---|---|
|user_id|TEXT PK FK|用户|
|used_on|TEXT PK|日期，YYYY-MM-DD|
|count|INTEGER|当天已用次数|
